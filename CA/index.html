Web Protocols

HTTP:

Hyper text Transfer protocol - HTTP is the method used to send packets over the internet

HTTPS:

Hyper text transfer protocol secure - The S means that the connect between the server and the client is encrypted meaning noone can see what was content was requested and recevied

TCP/IP:

TCP Transmision control protocol - Is the protocol for how data is broken down into smaller packets to be sent over the web and how it is rebuilt into the original file. Internet Protocol - is a unique string of numbers that routers and servers have like an address and is used to guide and keep the packets together to get to their destination

FTP:

File transfer protocol is the method of uploading and downloading files across the web

The Client-Server Model

The client sends a request to a server requesting a specific page or data. The server validates this request and then retrives the data and sends it to the client. The client downloads the page and then displays it to the user using the browser

Request-Response Model

The request response model is how a client sends a TCP/IP request to the server requesting data and then the server respondes with the information which is sent in small packets that the TCP breaks down and the IP helps guide the packets to its destination.
